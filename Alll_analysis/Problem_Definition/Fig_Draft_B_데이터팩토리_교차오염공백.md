# 문제정의서 초안 B (Fig 벤치마킹) — 관점: 데이터 신뢰 기반 붕괴 (교차오염 정보의 체계적 부재)

> **작성일:** 2026-04-11
> **벤치마크 경쟁사:** Fig (Food is Good)
> **시장 영역:** 식품 성분 데이터 신뢰 인프라 & 알러지 정보 서비스 시장
> **관점 키워드:** NLP 데이터 팩토리 / 교차오염 미공개 정보 / 데이터 공급자 종속 탈피

---

## 산업과 시장영역 분석

(공공 식품 안전 데이터 산업 / 알러지 성분 신뢰 정보 서비스 시장)에 대한 우리의 분석 결과 이 영역은

1) (기초 리서치) 시장의 거시적 트렌드 및 기본 양상이 '식약처 공공 API는 법적 표시 의무 성분만을 제공하며, 제조 공정에서 발생하는 교차오염 경고(예: "호두를 사용한 설비에서 제조") 및 복잡한 특수 식단 규칙(Low FODMAP, Vegan, 글루텐 프리 등)은 비정형 텍스트로 분산되어 체계적으로 수집·가공되지 않는 상태'이고, 이 원시 데이터를 NLP로 가공하는 자체 데이터 팩토리 역량을 확보한 사업자만이 공급자 종속에서 벗어나 진정한 데이터 초격차를 구축할 수 있으며,

2) (Porter 분析) 경쟁도가 낮고 브랜드 충성도가 높으며, 정밀 데이터 파이프라인 구축 비용이 신규 진입자에게 사실상 극복 불가능한 기술 장벽으로 작용함. 공급자(식약처 API·데이터 SaaS 기업) 교섭력이 현재로선 높으나, 자사 데이터 팩토리와 누적 유저 데이터를 확보한 사업자는 역으로 제조사에 소비 트렌드 데이터를 역제공하는 바터(Barter) 관계로 전환되어 교섭력이 역전된다.

3) (Value Chain 분析) 핵심 가치 창출 구조는 '비정형 라벨 텍스트·제조사 홈페이지·크롤링 데이터 → NLP 파싱 → 교차오염·특수식단 룰셋 DB화(데이터 팩토리)', '임상영양사·데이터 아키텍트 코어팀 → 의학 기반 교차 검수 → 소비자 신뢰 확보 브랜드 구축', '누적 익명 스캔 트렌드 데이터 → 식품 제조사 B2B 역제공 → API 비용 상계 + DaaS 수익화 파이프라인 형성' 이다.

## 해결하고자 하는 문제

따라서,
[특정 알러지나 특수 식단을 엄격히 관리해야 하는 알러지 환아 보호자 및 식이 제한 소비자]가
[마트·온라인몰에서 식품을 선택하는 의사결정 상황]에서 겪는
[법적 표시 라벨 이면에 존재하는 비정형 교차오염 경고 및 복잡한 식단 규칙 정보를 신뢰할 수 있는 형태로 즉각 조회할 수단이 없어 발생하는 치명적 정보 공백과 판별 불능의 불안]을
해결하는 것이 중요한 문제이다.

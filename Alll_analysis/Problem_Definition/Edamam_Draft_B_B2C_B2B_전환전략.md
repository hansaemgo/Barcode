# 문제정의서 초안 B (Edamam 벤치마킹) — 관점: B2C 트래픽 → B2B 데이터 권력 전환 전략 부재

> **작성일:** 2026-04-11
> **벤치마크 경쟁사:** Edamam (B2B DaaS 모델)
> **시장 영역:** B2C 앱→B2B API 확장 수익화 시장 / 데이터 가공 무역(Data Refinery) 시장
> **관점 키워드:** B2C→B2B 전환 / 데이터 가공 무역 / 트래픽 레퍼런스 기반 B2B 영업

---

## 산업과 시장영역 分析

(소비자 식품 스캐닝 앱 산업 / B2C 트래픽 기반 B2B 알러지 데이터 솔루션 확장 시장)에 대한 우리의 분석 결과 이 영역은

1) (기초 리서치) 시장의 거시적 트렌드 및 기본 양상이 '국내외 성공한 식품·헬스케어 앱들은 B2C 소비자 시장에서 트래픽과 데이터를 먼저 증명한 뒤, 이를 레퍼런스 삼아 B2B API 공급자로 지위를 전환하여 영속 수익을 확보하는 "데이터 가공 무역(Data Refinery)" 전략을 취하고 있으나', 국내 알러지 헬스케어 앱 시장에서는 이 두 단계를 통합적으로 설계하며 진입하는 사업자가 전무하여, B2C 단계에서 수익 모델 부재로 조기 소멸하거나 B2B 확장 없이 성장이 정체되는 패턴이 반복되고 있으며,

2) (Porter 分析) 경쟁도가 낮고 브랜드 충성도가 높으며, B2C 앱으로 맘카페·커뮤니티 바이럴을 통해 '유아 알러지 필수 앱'으로 입지를 굳힌 사업자만이 B2B 기업 고객 영업 시 "실사용 트래픽 레퍼런스"를 무기로 신뢰를 즉시 획득하는 비대칭 경쟁 우위를 보유함. B2B API가 기업 시스템에 한 번 삽입되면 전환 비용은 사실상 무한대이며, 이 단계에서 후발 경쟁자는 진입 자체가 불가능한 구조다.

3) (Value Chain 分析) 핵심 가치 창출 구조는 'B2C 앱 맘카페 바이럴 → 트래픽·데이터 입증 → 식품 제조사·커머스 개발팀 영업 레퍼런스 확보', '누적 유저 익명 스캔 트렌드 데이터 무기화 → 식품 제조사 소비 패턴 B2B 역제공(Barter) → API 비용 상계 + 데이터 DaaS 수익화', 'B2C Freemium 구독 수익(1차) + B2B API 종량제 ARR(2차) + 데이터 역판매(3차) 삼층 수익 파이프라인 완성' 이다.

---

## 해결하고자 하는 문제

따라서,
[알러지 헬스케어 앱 시장에 진입하는 신규 스타트업 창업자 및 사업기획자]가
[B2C 앱 출시 이후 지속 성장과 수익화 경로를 설계해야 하는 비즈니스 모델 전략 수립 상황]에서 겪는
[B2C 트래픽 확보 단계와 B2B 데이터 API 수익화 단계를 통합적으로 연결하는 전략적 설계 없이 B2C 단독 Freemium만으로는 영속 수익이 불가능하다는 수익 구조의 구조적 한계]를
해결하는 것이 중요한 문제이다.
